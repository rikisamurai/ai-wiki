[[harness-engineering|Harness Engineering]] 一个反直觉的核心结论：**赋予 Agent 更高的自主性，需要的是更严格的约束，而不是更宽松的约束。**

> [!important] 自由 ≠ 潜力
> 人们往往认为给 AI 更多自由能激发更大潜力。但实践表明，缺乏约束的 Agent 不是在"自由发挥"，而是在随机漫步。边界越清晰，Agent 在边界内的执行越可靠。

**Harness 是被动反应式生长的，不是预先设计的。** 正确的工作方式是：每当 Agent 犯了某类错误，就把"让这个错误永远不再发生"这件事工程化——写成测试、Lint 规则或 CI 检查。Harness 通过积累失败经验逐渐生长，而非项目开始时一次性预设完整。

> [!example] 比喻
> Harness 不是盖房前画好的图纸，而是每次漏水后加装的防水层、每次火灾后加装的烟雾报警器——是对现实摩擦力的工程化回应。

判定信号见 [[harness-engineering]] 的 Harness Mode：当 Agent 犯错，你的第一反应是"修这个错误"还是"工程化一个约束让它永不再发生"。后者就是约束悖论的实践姿态。
